Jotai vs. Redux

Jotai is very different from Redux and React Context API in almost every way. But there's one central concept that is the catch-allâ€”the one that you need to internalize. Redux stores are monolithic, but Jotai is atomic. This means, in Redux, it's a pattern to store all the needed global state in the app in one single big object. In Jotai, itâ€™s the opposite. You break your state into atoms i.e. - one store for one single store, or for closely related state.


Getting started with Jotai

Installing Jotai

# pnpm
pnpm add jotai

# npm
npm install jotai

# Or if you're a yarn person
yarn add jotai

Setting it up in the app

Jotai requires a provider to be present in the parent of the current component where it is being used. The simplest way to do would be to wrap the whole app in the provider as shown below ðŸ‘‡

// index.jsx (or index.tsx)
import React from 'react';
import ReactDOM from 'react-dom';
import { App } from './App';

// Jotai provider
import { Provider } from 'jotai';

ReactDOM.render(
  <React.StrictMode>
    <Provider>
      <App />
    </Provider>
  </React.StrictMode>,
  document.getElementById('root'),
);

Basic syntax

Creating your first atom

import { atom } from 'jotai';
const themeAtom = atom('light');

And that's it! You have your very first piece of state in!!


Now, how do you use it? Well, using it is a cross between the useState and useContext hooks. 

import { useAtom } from 'jotai';

export const ThemeSwitcher = () => {
  const [theme, setTheme] = useAtom(themeAtom);

  return <main>{theme}</main>;
};

Exactly like useState, but the only difference is that the atom we created is to be passed to useState. useAtom returns an array of size 2, where the 1st element is a value and the 2nd element is a function, to set the value of the atom. This makes all the components relying on this atom update and re-render. So, if we put it all together, the complete code would look like this ðŸ‘‡ 

import { atom, useAtom } from 'jotai';

const themeAtom = atom('light');

export const ThemeSwitcher = () => {
  const [theme, setTheme] = useAtom(themeAtom);

  return <main>{theme}</main>;
};

And notice the setTheme isn't used yet. Let's change that ðŸ‘‡ 

import { atom, useAtom } from 'jotai';

const themeAtom = atom('light');

export const ThemeSwitcher = () => {
  const [theme, setTheme] = useAtom(themeAtom);

  return (
    <main>
      <p>Theme is {theme}</p>

      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>Toggle Theme</button>
    </main>
  );
};

Conclusion

Simplicity: Jotai provides a simple and intuitive API for managing state in React applications. It uses atoms as the building blocks for state, making it easy to create, update, and access state in a straightforward manner. Performance: Jotai is designed to be lightweight and efficient, which can lead to better performance compared to other more complex state management solutions. It aims to minimize unnecessary re-renders and optimize the update process. Scalability: Jotai is scalable and can be used for managing both simple and complex state structures in React applications. It allows you to compose atoms together to create more intricate state dependencies. Flexibility: Jotai offers flexibility in how you structure and manage state in your application. It allows you to define state at the atom level and compose them in a way that fits your specific requirements. Ease of Integration: Jotai can be easily integrated into existing React applications without introducing a lot of overhead. It can coexist with other state management solutions or libraries, giving you the flexibility to use it where it makes sense.

Overall, Jotai can be a good choice for state management in React applications due to its simplicity, performance, scalability, flexibility, and ease of integration.
